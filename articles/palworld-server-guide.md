## 为什么自建专用服务器

幻兽帕鲁支持多人联机，你可以通过 Steam 好友邀请直接加入别人的世界，但这种方式有一个致命问题：**房主退出，全员掉线**。

专用服务器（Dedicated Server）是一台 7×24 小时运行的独立服务器，不依赖任何玩家的客户端。它的好处是：

- **全天在线**：任何人随时可以登录，不受房主是否在线的限制
- **性能更好**：服务器独立运行，不占用游戏客户端资源
- **可自定义**：经验倍率、捕获率、刷新率等参数完全由你控制
- **数据安全**：存档在你的服务器上，定期备份，不怕丢失
- **人数更多**：官方客户端联机最多 4 人，专用服务器最多支持 **32 人**

## 第一步：选购云服务器

### 配置推荐

幻兽帕鲁服务器对**内存**和**CPU 单核性能**要求较高。以下是不同玩家规模的配置建议：

| 玩家数量 | CPU | 内存 | 系统盘 | 带宽 | 月费用参考 |
|----------|-----|------|--------|------|-----------|
| 2-4 人（朋友联机） | 4 核 | 16 GB | 50 GB SSD | 5 Mbps | ¥70-120 |
| 4-16 人（小型公会） | 4-6 核 | 32 GB | 80 GB SSD | 10 Mbps | ¥150-300 |
| 16-32 人（大型公会） | 8 核 | 64 GB | 100 GB SSD | 20 Mbps | ¥400-800 |

**关键提醒：**

- **内存是第一瓶颈**：每增加一名在线玩家，大约需要额外 500MB-1GB 内存。8GB 内存基本跑不动，16GB 是起步线
- **必须用 SSD**：机械硬盘会导致严重的卡顿和加载延迟
- **CPU 看单核**：帕鲁服务器对单核性能敏感，高主频比多核更重要。优先选 3.5GHz+ 的 CPU
- **带宽**：4-8 人联机 5Mbps 够用，16 人以上建议 10Mbps+

### 云厂商对比

| 云厂商 | 优势 | 适合人群 | 入口 |
|--------|------|----------|------|
| **腾讯云** | 国内延迟低，轻量服务器性价比高，学生优惠力度大 | 国内玩家首选 | cloud.tencent.com |
| **阿里云** | 机型丰富，ECS 稳定性好，新用户首购优惠大 | 追求稳定 | aliyun.com |
| **棉花云** | 专注游戏云服务器，价格便宜，开箱即用 | 预算有限的玩家 | yun.88sup.com |

> **省钱技巧**：
> - 新用户首购优惠通常力度最大，建议直接买 1 年
> - 轻量应用服务器比云服务器 ECS 便宜，对游戏服务器来说足够了
> - 学生认证可以拿到更低的价格（腾讯云、阿里云都有学生机）
> - 关注双11、618 等大促活动

### 操作系统选择

**推荐：Ubuntu 22.04 LTS** 或 **Debian 12**

脚本支持 Ubuntu 22.04+ 和 Debian 11+，选这两个系统可以保证一键脚本顺利运行。CentOS 7 已经停止维护，不建议使用。

## 第二步：连接服务器

购买完云服务器后，你会获得一个**公网 IP** 和 **root 密码**（或密钥文件）。

### Windows 用户

推荐使用 [MobaXterm](https://mobaxterm.mobatek.net/) 或 [Tabby](https://tabby.sh/)，输入 IP 和密码即可连接。

也可以使用 Windows 自带的 PowerShell：

```bash
ssh root@你的服务器IP
```

### macOS / Linux 用户

直接在终端中：

```bash
ssh root@你的服务器IP
```

首次连接会提示确认指纹，输入 `yes` 即可。

## 第三步：一键部署

### 下载脚本

连接到服务器后，执行以下命令下载部署脚本：

```bash
wget -O palworld-server-install.sh https://raw.githubusercontent.com/你的用户名/game-server-scripts/main/palworld-server-install.sh
```

或者，如果你是直接在服务器上操作，可以用 `scp` 从本地上传：

```bash
scp palworld-server-install.sh root@你的服务器IP:/root/
```

### 运行安装

```bash
chmod +x palworld-server-install.sh
sudo ./palworld-server-install.sh
```

脚本启动后会先检测系统环境，然后展示一个配置菜单：

```
  ╔══════════════════════════════════════════════════════════╗
  ║          幻兽帕鲁服务器配置                              ║
  ╚══════════════════════════════════════════════════════════╝

  ┌─────────────────────────────────────────────┐
  │  1) 服务器名称:  Palworld Server            │
  │  2) 游戏端口:    8211                        │
  │  3) 最大玩家数:  32                          │
  │  4) 服务器密码:  (无密码)                    │
  │  5) 管理员密码:  admin123                    │
  │  6) 经验倍率:    1.0                         │
  │  7) 帕鲁捕获率:  1.0                         │
  └─────────────────────────────────────────────┘

  回车使用默认 / 输入 c 自定义:
```

- **直接按回车**：使用全部默认配置，适合快速体验
- **输入 `c` 然后回车**：逐项自定义配置

### 自定义配置

如果选择自定义，脚本会依次询问：

| 配置项 | 说明 | 建议值 |
|--------|------|--------|
| 服务器名称 | 显示在服务器列表中的名字 | 自定义 |
| 游戏端口 | 客户端连接端口 | 8211（默认即可） |
| 最大玩家数 | 同时在线人数上限 | 根据内存定：16G 选 16，32G 选 32 |
| 服务器密码 | 设密码后需要密码才能加入 | 朋友玩建议设密码 |
| 管理员密码 | 用于 RCON 远程管理 | 设一个强密码 |
| 经验倍率 | 1.0 是官方默认，2.0 是双倍 | 休闲玩家建议 2.0-3.0 |
| 帕鲁捕获率 | 影响捕捉成功率 | 休闲玩家建议 1.5-2.0 |

配置完成后，脚本会显示部署步骤概览，输入 `y` 确认开始安装：

```

  [5] 创建运行用户
  [6] 下载帕鲁服务器
```

整个安装过程大约需要 **10-20 分钟**，取决于服务器带宽和 SteamCMD 下载速度。

## 第四步：配置安全组

**这一步很多新手会遗漏，导致游戏连不上！**

脚本已经自动配置了服务器系统的防火墙（iptables/ufw），但云服务器还有一层**安全组**（也叫防火墙规则），需要在云厂商控制台手动添加。

### 以腾讯云为例

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 进入 **轻量应用服务器**（或云服务器 CVM）
3. 点击你的服务器实例
4. 进入 **防火墙** → **添加规则**
5. 添加以下规则：

| 协议 | 端口 | 策略 | 来源 |
|------|------|------|------|
| UDP | 8211 | 允许 | 0.0.0.0/0 |
| UDP | 27015 | 允许 | 0.0.0.0/0 |
| TCP | 25575 | 允许 | 0.0.0.0/0（可选，RCON） |

### 以阿里云为例

1. 登录 [阿里云控制台](https://ecs.console.aliyun.com/)
2. 进入 **ECS 实例详情**
3. 点击 **安全组** → **配置规则**
4. **入方向** → **手动添加**
5. 同上添加 UDP 8211 和 UDP 27015

> **为什么要配两次防火墙？**
> 
> Linux 系统自带一层防火墙（iptables/ufw），脚本会自动放行端口。但云服务器的宿主机还有一层安全组，这是云厂商提供的网络层面的防火墙。两层都放行，外部才能正常访问。

## 服务器管理

脚本会自动安装 `pal-manager` 管理工具，让日常运维变得简单。

### 常用管理命令

```bash
# 查看帮助
pal-manager

# 服务器控制
pal-manager start          # 启动服务器
pal-manager stop           # 停止服务器
pal-manager restart        # 重启服务器
pal-manager status         # 查看运行状态

# 监控与调试
pal-manager logs           # 实时查看日志 (Ctrl+C 退出)
pal-manager players        # 查看在线玩家
pal-manager memory         # 查看内存使用情况
pal-manager info           # 显示服务器信息（IP、端口等）

# 管理操作
pal-manager broadcast "服务器将在5分钟后重启"  # 广播消息给所有玩家
pal-manager backup         # 立即备份存档
pal-manager update         # 更新服务器到最新版本
pal-manager config         # 编辑配置文件（修改后需重启生效）
```

### 修改服务器配置

```bash
pal-manager config
```

这会用 nano 编辑器打开配置文件。修改完成后保存退出，然后重启服务器：

```bash
pal-manager restart
```

**常用配置项：**

```ini
# /opt/palworld/PalWorldSettings.ini

Difficulty=Normal              # 难度: Normal / Hard
DayTimeSpeedRate=1.000000      # 白天时间流速 (越大白天越长)
NightTimeSpeedRate=1.000000    # 夜晚时间流速
ExpRate=1.000000               # 经验倍率
PalCaptureRate=1.000000        # 帕鲁捕获率
PalSpawnNumRate=1.000000       # 帕鲁刷新数量倍率
CollectionDropRate=1.000000    # 采集掉落倍率
WorkSpeedRate=1.000000         # 工作速度倍率
bEnablePlayerToPlayerDamage=False  # 玩家间伤害
```

### 更新服务器版本

当游戏发布新版本时，执行：

```bash
pal-manager update
```

脚本会自动停止服务器、下载更新、重新启动。更新过程中会自动备份存档，不用担心数据丢失。

## 性能优化与调优

脚本已经内置了多项性能优化，但你还可以根据实际情况进一步调整。

### 内存优化

如果服务器内存紧张（16GB），可以降低内存限制：

```bash
# 编辑 systemd 服务文件
sudo nano /etc/systemd/system/pal-server.service

# 修改这两行（根据你的内存调整）
MemoryMax=12G
MemoryHigh=10G

# 重载配置
sudo systemctl daemon-reload
sudo systemctl restart pal-server
```

### 内核参数优化

脚本已经自动优化了以下内核参数，一般不需要手动调整：

```
vm.max_map_count = 262144    # 帕鲁服务器要求至少 262144
vm.swappiness = 10           # 减少 swap 使用，优先使用物理内存
fs.file-max = 65535          # 最大文件描述符数
net.core.somaxconn = 4096    # 网络连接队列长度
```

### Swap 交换空间

对于 16GB 内存的服务器，脚本会自动创建 16GB 的 Swap 文件。这在内存不足时可以防止服务器被 OOM Killer 杀掉。如果你的内存充足（32GB+），可以适当减小 Swap。

### 自动重启

脚本配置了每天凌晨 4:00 自动重启服务器。这是为了防止内存泄漏导致的性能下降。帕鲁服务器运行时间越长，内存占用越高，定期重启是必要的。

如果需要修改重启时间：

```bash
sudo nano /etc/systemd/system/pal-restart.timer
# 修改 OnCalendar= 行
sudo systemctl daemon-reload
sudo systemctl restart pal-restart.timer
```

## 常见问题排查

### 连接超时

**症状**：客户端输入 IP 后一直转圈，最终提示连接超时。

**排查步骤**：

```bash
# 1. 检查服务器进程是否在运行
pal-manager status

# 2. 检查端口是否在监听
ss -ulnp | grep 8211

# 3. 检查系统防火墙
sudo iptables -L -n | grep 8211
# 或
sudo ufw status

# 4. 检查云安全组是否放行了 UDP 8211（在云控制台查看）
```

最常见原因：**云安全组没有放行 UDP 8211**。系统防火墙脚本会自动配置，但安全组需要在云厂商控制台手动添加。

### 服务器启动失败

```bash
# 查看详细错误日志
pal-manager logs

# 或
sudo journalctl -u pal-server -n 50 --no-pager
```

常见原因：
- 内存不足 → 检查 `free -h`，确认 Swap 是否启用
- SteamCMD 下载不完整 → 运行 `pal-manager update` 重新下载
- 端口被占用 → `ss -ulnp | grep 8211` 查看是否有其他进程占用

### 游戏卡顿

- 检查 CPU 使用率：`htop` 或 `top`
- 检查内存使用：`pal-manager memory`
- 检查磁盘 I/O：`iostat -x 1`
- 如果 CPU 持续 100%，考虑减少最大玩家数或升级配置

### 存档丢失

脚本配置了每 6 小时自动备份，备份文件在：

```bash
ls -lh /opt/palworld/backups/
```

恢复备份：

```bash
pal-manager stop
cd /opt/palworld
tar -xzf backups/pal_backup_XXXXXXXX_XXXXXX.tar.gz
pal-manager start
```

## 脚本做了什么

如果你好奇一键脚本到底做了哪些事情，以下是完整的技术细节：

### 1. 内核参数优化

修改 `/etc/sysctl.conf`，设置 `vm.max_map_count=262144`、`vm.swappiness=10` 等参数。帕鲁服务器要求 `vm.max_map_count` 至少为 262144，否则会启动失败。

### 2. Swap 交换空间

检测物理内存，如果小于 32GB 则自动创建等大的 Swap 文件。使用 `fallocate` 创建，优先级设置为 10。这可以防止内存不足时服务器被 OOM Killer 杀掉。

### 3. 安装依赖

安装 `curl`、`wget`、`lib32gcc-s1`、`screen` 等依赖组件，以及 32 位运行库（SteamCMD 需要）。

### 4. 安装 SteamCMD

优先使用 apt 安装，如果不可用则从 Valve 官方下载。

### 5. 创建专用用户

创建 `steam` 系统用户，服务器以非 root 权限运行，提高安全性。

### 6. 下载服务器

通过 SteamCMD 匿名登录下载帕鲁服务器（AppID: 2394010），带重试机制。使用 `+force_install_dir` 确保文件安装到正确目录。

### 7. 生成配置文件

根据用户选择的参数生成 `PalWorldSettings.ini`，包含服务器名称、密码、经验倍率等所有配置项。

### 8. 创建 systemd 服务

配置自动启动、自动重启、内存限制（MemoryMax/MemoryHigh）、OOM 保护、安全加固（ProtectSystem、PrivateTmp、NoNewPrivileges）。

### 9. 自动任务

- **每日凌晨 4:00 自动重启**：防止内存泄漏
- **每 6 小时自动备份**：保护存档数据，最多保留 20 份

### 10. 防火墙配置

自动配置 iptables/ufw，放行 UDP 8211、UDP 27015、TCP 25575。
